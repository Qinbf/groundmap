# E5: Text Embeddings by Weakly-Supervised Contrastive Pre-training ^h-1-1-99ca94

> **原始文件**: [[raw/papers/2022-12-e5]]
> **作者**: Liang Wang, Nan Yang et al.(Microsoft)
> **发表**: 2022-12 arXiv preprint(EACL 2023)
> **arXiv**: [2212.03533](https://arxiv.org/abs/2212.03533)
> **历史地位**:**LangChain / LlamaIndex 默认 embedding 之一**;E5-large-v2 是 2023 年最广泛使用的开源 embedding model ^p-1-974b0d

## 摘要 ^h-2-1-3ae146

E5 提出 **CCPairs**(Contrastive Pre-training pairs)— **270M 大规模 text pair 数据集**,源自 Reddit / StackExchange / 各种 web 来源。**两阶段训练**:weakly-supervised 在 CCPairs 上预训练,再在 MS MARCO / NQ 等 supervised 数据上 fine-tune。E5 在 **MTEB benchmark** 上 SOTA(2022 末),且**通用性强**——同一模型用于 retrieval / clustering / classification / pair classification 等多 task[[raw/papers/2022-12-e5#^h-2-4-18931e]]。 ^p-2-65eaa5

## 关键发现 / 论点 ^h-2-2-10e39a

1. **数据规模 = embedding 质量**:CCPairs 270M 显著大于 Contriever 的 wiki pair 数据[[raw/papers/2022-12-e5]]
2. **两阶段训练范式**确立:大规模 weak supervision pre-train → 小规模 supervised fine-tune [KB 综合]
3. **MTEB 是 embedding 评估的事实标准**(本论文推动)— 后续 BGE / NV-Embed 都在 MTEB 上对比 [[raw/papers/2022-12-e5]]
4. **E5-small / base / large 三种规模**,工程实用性极强(部署成本可选) [[raw/papers/2022-12-e5]] ^p-3-8d5f19

## AI 综合判断 ^h-2-3-e0eacb

### 核心价值 ^h-3-1-57ae37

E5 让 dense embedding 从"研究工件"变成**工程默认选项**——LangChain / LlamaIndex / Haystack 等 RAG 框架的开箱即用 embedding。它的两阶段训练范式(weak → supervised)成为后续 BGE / GTR / NV-Embed 等工作的共同方法论。 [KB 综合] ^p-4-f8c2fb
