⚠️ 传统 chunk-RAG 会怎样

1. 把 4 个 source 切成 200-500 token chunks → 大概 60-80 个片段

2. 用 embedding 找 top-K（通常 K=5）→ 返回 5 个语义相似 chunk

3. 把 5 个 chunk 塞给 LLM 让它综合 → 缺乏 concept / analysis 这种层级聚合

4. 引用粒度 = chunk（含上下文噪声）→ 用户无法精确反查到段

5. 失败模式：如果 5 个 chunk 全来自同一个 source（同质化召回），就丢失了多源印证；如果跨源但语义噪声大，综合答案不连贯
