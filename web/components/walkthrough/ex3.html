  • 2.71 万亿元 +14%（海关总署，最终核算，2025-10 发布） [[raw/papers/mofcom_digital_trade_2025#^p-617-26618b]]

使用建议

  • 需要绝对值（合同 / 投资估值）→ 用 2.71 万亿（最终核算，方法学一致）
  • 需要增速对比（2024 vs 2025）→ 用 商务部 +10.8%（与 2025 同口径可比）

关键宏观信号（顺带提）

2025 增速从 +14% / +10.8% 腰斩到 +6.4%，时间窗与美国 De Minimis 取消高度吻合。建议跟踪 2026 H1 数据看反弹。

⚠️ 传统 chunk-RAG 会怎样

1. embedding 召回大概率把两个 chunk（2.63 vs 2.71）都拿到

2. LLM 看不到结构语义（哪个是商务部 / 哪个是海关 / 哪个是快报 / 哪个是最终）— 只能从 chunk 内文字里推

3. 多数模型会"折中"：报"约 2.65-2.7 万亿"——两边都不准、还掩盖了官方部门差异

4. 缺少"使用建议"维度：哪个口径用于绝对值 / 哪个用于增速对比——这是用户真正要的指导
