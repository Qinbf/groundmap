# Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks ^h-1-1-a7c310

我们提出 **检索增强生成（Retrieval-Augmented Generation，RAG）**—— 一种把**预训练参数化记忆**（一个 seq2seq 模型 BART）与**非参数化记忆**（用神经检索器 DPR 访问的维基百科稠密向量索引）结合起来的通用微调范式。在开放域问答（NaturalQuestions、TriviaQA、WebQuestions）上 RAG 刷新 SOTA，并且生成的语言比纯参数化的 BART **更具体、更真实、更少幻觉**。 ^p-1-3e8c92

## Abstract ^h-2-1-b4d217

大型预训练语言模型把事实知识隐式地存进**参数**里，在下游任务上表现强劲。但这种"参数化记忆"有两个根本问题：**难以检视与更新**（要改一条事实就得重训），并且在知识密集型任务上**倾向于幻觉**——给出听起来流畅、实则编造的答案。 ^p-2-7f1a08

我们提出 RAG —— 一个把参数化与非参数化记忆结合的通用微调配方。RAG 在三个开放域问答任务（NaturalQuestions、TriviaQA、WebQuestions）上达到 SOTA，**超过了抽取式（extractive）方法与纯参数化 seq2seq 基线**；在抽象式生成任务上，RAG 产出的文本比 BART 更真实、更具体、更多样。 ^p-3-bd4e71

## Introduction ^h-2-2-c0712f

预训练语言模型（BERT、GPT、BART）在参数中学到了大量事实知识 —— 这被称为**参数化记忆（parametric memory）**。但它是隐式的：**无法被检视、无法被定点更新**，而且当被问到训练语料里没充分覆盖的事实时，模型会**幻觉**。 ^p-4-2c8a91

带**非参数化记忆**（基于检索的记忆）的模型可以缓解这些问题：知识被显式存在一个可检索的语料里，**可以直接增删改**，并且模型用到的知识**能被指出来源**（可溯源）。但此前的混合模型大多是任务专用的，且只在抽取式场景里验证过。 ^p-5-c891ef

REALM 与 ORQA 最近把"掩码语言模型 + 可学习检索器"结合起来，但它们**只探索了抽取式的开放域问答** —— 答案是从检索到的段落里抠出一段 span，而不是生成。 ^p-6-49b3c0

本文把这种**参数化 / 非参数化混合记忆**带到**生成**任务上 —— 这就是 RAG。其中参数化部分是预训练 seq2seq 模型（BART），非参数化部分是一个**维基百科稠密向量索引**，由预训练神经检索器（DPR）访问。 ^p-7-8d7e62

## Model Architecture ^h-2-3-d4b201

RAG 模型用输入序列 x 去**检索文本文档 z**，再把 z 作为生成目标 y 时的**额外上下文**。它由两个组件构成：**检索器** p_η(z | x)，返回与 x 最相关的 top-K 段落；**生成器** p_θ(y_i | x, z, y_{1:i-1})，基于原输入、检索到的文档与已生成前缀产出下一个 token。 ^p-8-1be4a3

为了端到端训练，我们把"检索到的文档"当作一个**隐变量（latent variable）**，并对它做**边缘化（marginalize）**。边缘化的方式有两种 —— 对应下文的 **RAG-Sequence** 与 **RAG-Token**。 ^p-9-7c305d

### Retriever 与 Generator ^h-3-1-4f8e29

**检索器**采用 **Dense Passage Retriever（DPR）**，是一个 bi-encoder：用 BERT_d 编码文档、BERT_q 编码 query，相关性 p_η(z|x) ∝ exp(d(z)^T q(x))。top-K 通过 **MIPS（最大内积搜索，用 FAISS）**高效求解。文档索引把整个维基百科切成约 **2100 万**个 100 词的段落（chunk）。 ^p-10-26b7f8

**生成器**采用 **BART-large（约 400M 参数）**，一个预训练的 seq2seq 去噪自编码器。生成时，把检索到的文档 z 与原输入 x **拼接**起来作为 encoder 的输入，decoder 自回归产出答案。 ^p-11-9af3b8

### 检索机制：把检索当隐变量 ^h-3-2-3d6f0a

**核心思路：不要求"先检索对、再生成"这种硬流水线**，而是把检索结果作为隐变量、让生成对它求加权和（边缘化）。这样检索器与生成器可以**联合训练**，检索质量的好坏会通过生成的 loss 反传回来。 ^p-12-c7f203

具体地，对 top-K 个检索段落，按检索分 p_η(z|x) 作为权重，对生成概率求和。这里没有"哪个文档是对的"这种显式标注 —— 监督信号只有最终答案。 ^p-13-58e9b0

```
RAG-Sequence: p(y|x) ≈ Σ_{z∈top-k} p_η(z|x) · Π_i p_θ(y_i | x, z, y_{1:i-1})
RAG-Token:    p(y|x) ≈ Π_i Σ_{z∈top-k} p_η(z|x) · p_θ(y_i | x, z, y_{1:i-1})
``` ^c-14-7d2e8c

两种边缘化各有适用面：**RAG-Token** 在生成每个 token 时都能从**不同文档**取证据，适合"答案需要拼接多个段落事实"的情形；**RAG-Sequence** 用同一个文档生成整句，更简单，在"单个段落就够"时已经很强。实践中两者都显著优于纯参数化基线。 ^p-15-bc92a1

我们检索 **top-K（K=5 或 10）**个段落。K 越大召回越高，但拼接进 context 的开销也越大 —— **K 是召回与成本之间的关键旋钮**，超过某个点后收益递减。 ^p-16-5f3047

### RAG-Sequence vs RAG-Token ^h-3-3-72bef4

**RAG-Sequence** 把同一个检索文档用于生成完整序列：先采样/取 top-K 文档，每个文档独立生成完整答案，再按检索分加权。 ^p-17-d29b7e

```
RAG-Sequence 的 score：对每个候选答案 y，
  score(y) = Σ_{z∈top-k} p_η(z|x) · p_θ(y | x, z)
``` ^c-18-1ae04b

**RAG-Token** 允许在生成**每个 token** 时使用不同文档：每一步对 top-K 文档的预测分布做加权和，再走标准 beam search。这让一个答案能融合分散在多个段落里的事实。 ^p-19-8b4f2a

本文主实验里 RAG-Token 与 RAG-Sequence 各有胜负：在需要综合多段证据的题上 RAG-Token 略优，在事实集中于单段的题上 RAG-Sequence 略优；二者都稳定超过纯参数化与抽取式基线。 ^p-20-c4170d

### 训练（联合、无检索监督）^h-3-4-91a5d8

检索器与生成器**联合训练**，目标是最小化答案的**负边缘对数似然**，**没有任何"该检索哪篇文档"的直接监督**—— 检索好坏完全由下游答案的 loss 隐式驱动。 ^p-21-7e6c1f

```
loss = - Σ_(x,y) log p(y | x)   # p(y|x) 已对检索文档 z 边缘化
``` ^c-22-3f8b09

一个关键工程取舍：训练时我们**固定文档编码器 BERT_d 与整个索引**，只微调 **query 编码器 BERT_q 与生成器**。原因是：每次更新 BERT_d 都要对 2100 万段落**重建索引**，代价高昂；实验表明固定文档编码器**几乎不损失性能**。 ^p-23-4dba87

### 解码与索引热插拔 ^h-3-5-8a2b59

由于 RAG 的知识在**非参数化索引**里，而不是锁死在参数里，它带来一个独特能力：**测试时可以直接替换索引来更新世界知识，完全不需要重训模型**。 ^p-24-c0bf41

我们用一个直接实验展示这点：把维基百科索引从一个时间快照换成另一个时间快照，模型对"现任某职位是谁"这类时效问题的答案随之改变 —— **知识更新 = 换索引**。 ^p-25-d72a08

```
# 更新世界知识：无需 retrain，换 index 即可
index = build_faiss_index(wikipedia_2018)   # 旧
index = build_faiss_index(wikipedia_2020)   # 新 —— 模型答案随之更新
``` ^c-26-9bc481

解码上，RAG-Token 可用标准 beam search（转移概率对文档求和）；RAG-Sequence 需要 "Thorough Decoding / Fast Decoding" 两种近似，在精度与速度间取舍。 ^p-27-218fc6

这种**可热插拔、可溯源**的非参数化记忆，是 RAG 区别于"把知识塞进更大模型参数"路线的根本特征 —— 也是后续整条 RAG 演化路线的起点。 ^p-28-7a2e1f

## 为什么要检索（vs 纯参数化）^h-2-4-be4790

本节我们把"检索增强"与"纯参数化（把知识全压进参数、靠 scale 取胜）"沿几个维度对比 —— 这些维度解释了为什么知识密集型任务上检索是必要的，而不仅是锦上添花。 ^p-29-5ef2cd

我们考虑三方面诉求来动机混合记忆的使用： ^p-30-b3c8a1

一是**知识可更新性**（改一条事实要多大代价）；二是**可溯源性**（能否指出答案依据的具体段落）；三是**幻觉倾向**（在长尾事实上是否倾向编造）。 ^p-31-6e7f93

下表对比纯参数化模型与检索增强模型在这三方面（外加几个工程维度）的差异： ^p-32-d18b2e

| 维度 | 纯参数化（BART / T5 / GPT） | 检索增强（RAG） |
|---|---|---|
| 知识更新 | 需重训或继续预训练 | **换索引即可（热插拔）** |
| 可溯源 | ✗ 黑盒，无法指出依据 | **✓ 能指向具体 passage** |
| 幻觉倾向 | 高（长尾事实尤甚） | **低（有外部证据约束）** |
| 知识容量 | 受参数量上限 | 受语料库规模（可扩展） |
| 推理/泛化 | 强（参数里的模式） | 同样靠生成器，不损失 | ^t-33-cf4a82

**检索增强在"知识更新 / 可溯源 / 抗幻觉"三个维度全面占优**，代价是引入了检索器与索引这套额外基础设施。**作为附加好处**，因为答案明确依赖被检索的段落，RAG 的输出**天然可解释、可审计** —— 你能看到模型"读了哪几段才这么答"。 ^p-34-7d28f0

## Results ^h-2-5-09b3ea

在开放域问答上，**RAG 刷新 SOTA**：NaturalQuestions 达到 **44.5 EM**，并在 TriviaQA、WebQuestions 上超过抽取式（REALM、DPR）与纯参数化 seq2seq（T5）基线 —— 一个**生成式**模型在抽取式擅长的 benchmark 上反超，是当时的关键结果。 ^p-35-bf1c08

在抽象式问答与生成任务上，**RAG 生成的文本比 BART 更真实（factual）、更具体、更多样**，幻觉明显减少。人工评测中标注者更偏好 RAG 的输出。 ^p-36-c8b4ea

## Conclusion ^h-2-6-7da3e1

本工作提出了 **RAG —— 第一个把参数化与非参数化记忆结合用于知识密集型生成的通用范式**，用一个可微的检索器 + 预训练 seq2seq 生成器，对检索文档做边缘化以端到端训练。 ^p-37-4f29a6

RAG 在多个开放域问答任务上刷新 SOTA，生成更真实的语言，且**非参数化记忆可以在不重训的情况下更新**。这把"知识"从模型参数里解放出来，变成一个可维护、可溯源的外部组件。 ^p-38-2bc908

我们对检索增强模型的未来感到兴奋，并计划**从头联合预训练检索器与生成器**，以及把这一范式推广到更多知识密集型任务 —— 后续 GraphRAG、Self-RAG、HippoRAG 等一系列工作都建立在这条路线之上。 ^p-39-d70e4c
